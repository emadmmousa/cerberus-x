# Security Policy

## Supported use

Firebreak is for **authorized** security testing and research. Operators must have permission for every target.

## Reporting vulnerabilities

Email or open a **private** GitHub security advisory for the maintainers. Do not file public issues for exploitable flaws in Firebreak itself until a fix is available.

Include: affected version/commit, reproduction steps, impact.

## Secrets

Never commit `.env`, API keys, proxy passwords, or engagement evidence. Rotate any credential pasted into chat or tickets.
